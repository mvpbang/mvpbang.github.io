拷贝公钥到目标客户机
01、目标机不存在默认的key（authorized_keys）
ssh-copy-id  -i id_rsa.pub  user@ip  
注意：自动在目标机上添加~.ssh/authorized_keys,，前提是目标不存在authorized_keys
02、目标机存在默认的key
scp id_rsa.pub  root@ip   #拷贝
ssh root@ip              #登录
cat id_rsa.pub >>~.ssh/authorized_keys  #追加新的公钥
chmod 700 .ssh && chmod 600 .ssh/authorized_keys #安全限制

0、生成密钥对有密码

对key加密码，本质是防止key的泄露，服务器权限的丢失。虽然key加密码但是在我们认证的过程中，也是需要输入key密码。
服务端key私钥代理只需输入一次密码后续不在需要输入key密码，需要引入第三方keychain
01、keychain编译安装
https://github.com/funtoo/keychain
make
cp keychain /usr/bin/
注意：keychain,在控制端，也就是私钥存储的服务器上！
02、配置登录环境变量
vim ~.bash_profile
keychain id_rsa
. ~/.keychain/$HOSTNAME-sh
source  ~.bash_profile
03、认证key
ssh root@ip

#第一次，需要输入秘钥的密码（目的为了防止私钥泄露还能有密码保护）
后续登录，不在会提示输入秘钥密码（除非机器重启）
上图就是后续登录服务端，提示的信息，本质也是运行系统默认的ssh-agent
